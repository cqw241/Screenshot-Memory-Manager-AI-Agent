# Task 2.4 时间轴与详情 UI 设计

- 日期：2026-07-21
- Story：S02 核心曳光弹
- 状态：已实现并通过模拟器验收（commit `eb8081b`）
- 依赖：Task 2.1–2.3 已完成

## 1. 目标

把 Task 2.3 已持久化的截图卡片变成可浏览、可查看原图、可删除的用户界面：主页以时间轴为第一视觉层级，卡片按仓储既定顺序展示；点击卡片进入独立详情路由，查看原图与全部结构化字段；删除成功后返回并刷新时间轴。

本 Task 的验收结果应满足：

1. 时间轴卡片与本地数据库内容一致，并按事件时间倒序展示。
2. 卡片展示类型徽标、确定性标题、关键实体和缩略图。
3. 空态显示“试试导入一张截图”及明确的导入入口。
4. 详情页展示应用私有原图和全部已持久化字段。
5. 用户可在二次确认后删除卡片；删除不影响相册原图、应用私有原图或缩略图。

## 2. 范围

### 2.1 本次实现

- 时间轴优先的主页布局和紧凑导入入口。
- 时间轴加载、正常、空态和读取失败状态。
- 基于现有 `listTimeline()` 结果的日期分组和卡片展示。
- 独立 ArkUI 详情页面与 `cardId` 路由参数。
- 详情页加载、正常、卡片不存在和读取失败状态。
- 删除确认、删除执行、失败重试和成功返回刷新。
- 展示格式化逻辑的自动测试与模拟器交互验收。

### 2.2 本次不实现

- 修改 LLM 输出 Schema 或为数据库新增标题字段。
- 编辑卡片类型或实体字段。
- 搜索、收藏、动作按钮执行、全屏缩放图片或系统分享。
- 删除相册原图、应用私有原图或缩略图文件。
- 通知、元服务卡片或搜索结果的真实深链入口；本次只为这些入口建立可复用的 `cardId` 详情契约。

## 3. 已确认决策

1. **首页采用时间轴优先布局。** 导入入口收敛为顶部主按钮，卡片直接进入首屏；不再保留占据大量空间的临时验收结果面板。
2. **标题从现有数据确定性生成。** 不扩展理解与存储契约。标题固定映射为“讲座海报”“待取快递”“截止事项”“优惠券”“未分类截图”。
3. **详情使用独立 ArkUI 页面路由。** 首页只传 `cardId`，详情页通过仓储重新读取最新卡片。
4. **Task 2.4 包含删除。** 删除前二次确认；成功后返回并刷新。删除只作用于数据库记录。

## 4. 架构与职责

### 4.1 首页入口

`Index.ets` 继续负责启动门禁、配置向导、导入管线与主页数据状态。主页相关状态至少包括时间轴卡片、加载标记和脱敏错误信息。

主页在以下时机重新调用 `ScreenshotCardRepositoryPort.listTimeline()`：

- 首次进入主页；
- 页面从详情恢复可见；
- 导入并保存卡片成功。

首页不读取 SQL、JSON 或数据库版本，也不自行排序仓储结果。

### 4.2 详情页面

新增独立的 `CardDetail` ArkUI 页面并注册到页面清单。路由参数只包含非空字符串 `cardId`。详情页创建同一仓储实现并调用 `getById(cardId)`，避免传递可能过期或不完整的卡片对象。

详情页删除时调用 `deleteById(cardId)`。只有仓储确认删除成功后才返回；失败时保留当前页面和卡片内容，显示可重试错误。

### 4.3 展示格式化模块

新增不依赖 ArkUI 或数据库的纯展示模块，集中负责：

- 类型中文标签、标题、徽标色和占位符；
- 事件日期分组和导入日期格式化；
- 关键实体摘要；
- 建议动作与解析状态的中文标签。

页面只消费格式化结果，不重复散落类型判断与空值处理。该模块输入为 `ScreenshotCard`，不改变原始数据。

## 5. 数据流与展示规则

### 5.1 时间轴顺序与分组

仓储层已经保证：有事件时间的卡片按事件时间、创建时间和 ID 倒序；无事件时间的卡片排在其后并按创建时间和 ID 倒序。UI 原样保留输入顺序，只在相邻卡片的分组标签变化时插入日期标题。

有 `eventTime` 的卡片以该时刻在设备当前时区对应的本地日历日期作为分组标签。无 `eventTime` 的卡片统一进入“未识别事件时间”分组，并在摘要中显示导入日期。UI 不从 `timeText` 猜测缺失日期。

### 5.2 卡片内容

每张卡片展示：

- `thumbnailUri` 对应的缩略图；
- 类型徽标和确定性标题；
- 存在的关键实体，优先级为原始时间表达、地点、码号、金额；
- 建议动作摘要；没有动作时不显示动作行。

缺失字段直接省略，不显示虚构信息。未分类卡片使用中性标题和暖色警示徽标，但不把安全 fallback 表述为系统故障。

### 5.3 详情内容

详情页使用 `imageUri` 展示应用私有原图，保持完整比例并使用 `ImageFit.Contain`。下方展示：类型、确定性标题、导入时间、原始时间表达、事件时间、地点、码号、金额、建议动作和解析状态。缺失字段显示“—”，便于用户确认数据库中确实没有该实体。

## 6. 界面状态与交互

### 6.1 时间轴首页

- **加载中：** 首次读取时显示轻量加载反馈。
- **正常：** 顶部为标题、模型设置和紧凑导入按钮，下方为日期分组时间轴。
- **空态：** 显示“试试导入一张截图”、数据范围说明和主导入按钮，不绘制空时间线。
- **读取失败：** 显示脱敏错误类别、简短说明和“重新加载”；不把旧列表当作最新结果。
- **导入中：** 导入按钮禁用并显示处理中状态；已有时间轴仍可见。
- **导入成功：** 保存成功后重新查询，并用紧凑状态条说明新卡片已出现。

### 6.2 详情页

- 顶部提供明确的返回入口。
- 点击删除先显示二次确认，文案明确说明“只删除拾光中的卡片，不影响相册原图”。
- 删除进行中时按钮禁用，防止重复提交。
- 用户取消确认时不进行任何写操作。
- 删除成功后返回时间轴；首页恢复可见时重新查询。

## 7. 错误处理

- 路由参数缺失或为空：不调用仓储，显示“无法打开这张卡片”和返回入口。
- `getById()` 返回 `undefined`：显示“卡片不存在或已被删除”和返回入口。
- 仓储读取失败或数据损坏：显示现有脱敏错误类别，不展示半成品字段。
- 缩略图或原图解码失败：显示图片占位，仍保留可读取的结构化字段。
- 删除返回 `false`：视为卡片已不存在，提示后允许返回时间轴。
- 删除抛错：停留详情页，保留内容并允许重试。

任何错误路径都不得记录图片 URI、卡片实体正文或其他用户内容。

## 8. 测试与验收

### 8.1 自动测试

为纯展示模块增加 Hypium 测试，覆盖：

- 五种类型的中文标签和确定性标题；
- 关键实体的优先级、空值省略和无动作情况；
- 事件时间在设备时区下的日期分组；
- 无事件时间分组与导入日期；
- 输入顺序保持，不在 UI 层二次排序；
- fallback 解析状态与动作中文标签。

完整运行既有仓储与导入/理解测试，确认 Task 2.4 未破坏 Task 2.1–2.3。

### 8.2 模拟器验收

1. 空库启动，看到“试试导入一张截图”。
2. 导入一张样例，保存后时间轴自动出现对应缩略图、类型、标题和实体。
3. 准备有事件时间与无事件时间的多张卡片，顺序和分组与 `listTimeline()` 一致。
4. 点击卡片进入独立详情页，原图和全部字段与数据库一致。
5. 取消删除，卡片保持不变。
6. 确认删除，返回后卡片消失；相册原图不受影响。
7. 强制停止并重启应用，删除结果与剩余卡片保持一致。
8. 构造无效卡片 ID 或已删除卡片入口，页面安全返回且不崩溃。
