### Task 2.2 LLM 理解客户端 — [实现完成，真实模型单样例通过，四样例待补]

- 实际做法（与计划的差异）：新增深理解模块，从 `filesDir/imports` 私有原图按比例生成 JPEG 85 缩略图并保存到独立的 `filesDir/thumbnails`；长边最大 1024 px，小图不放大。Base64 Data URL 只在缩略图生成器与模型请求内部传递，页面只接收缩略图 URI、宽高和编码字节数。主页现于单图导入和端侧 OCR 后继续发起理解，即使 OCR 为空也保留多模态图片理解，并在临时验收区展示类型、实体、动作与解析状态；正式卡片仍留待 Task 2.3–2.5。
- 请求与解析：调用必须经过 `ConfigRepository.withVerifiedConfig(...)`，并复用既有 URL 预检、可信源校验与 HTTP transport。Prompt 注入同一次捕获的设备本地时间、UTC 偏移和系统时区，要求保留 `timeText`、不得猜测缺失日期，并固定返回原始 JSON 对象。客户端精确校验对象字段、nullable string、类型/动作枚举及带时区且日历有效的 ISO-8601 `eventTime`；Markdown 围栏、语法错误、缺失/额外字段和错类型均为内容非法。首次非法使用同一 OCR、缩略图和时间上下文纠错一次，且不携带第一次原始回复；二次非法返回 `UNCATEGORIZED + 全 null 实体 + 空 actions + FALLBACK_INVALID_RESPONSE`。
- 安全与错误：每次请求对象的 Key 在 `finally` 清空，运行时 Key 再由 `withVerifiedConfig` 清空；Key、Base64、OCR 正文、完整请求体和模型原始回复不进入理解结果或日志。401/403、404/405、429、网络/超时、图片能力拒绝、其他 4xx/5xx 与外层响应不兼容分别返回脱敏分类错误，不生成未分类结果；未实现指数退避、后台重试或持久化队列。生产日志只记录缩略图宽高、编码字节数、请求次数、耗时与错误类别。

### Task 2.3 卡片存储层 — [实现完成，设备重启验收通过]

- 实际做法：新增卡片深模块，页面只通过 `create/getById/updateUnderstanding/deleteById/listTimeline` 接口访问卡片，不接触 SQL、JSON 编解码和数据库版本。卡片 ID 复用应用私有原图 ID，持久化私有原图 URI、缩略图 URI、类型、精确实体 JSON、动作 JSON、解析状态、创建时间及事件时间索引；不存 OCR 正文、Base64、API Key、模型原始回复或完整请求体。
- RelationalStore：生产适配器使用加密的 `shiguang_cards.db`、`SecurityLevel.S3` 和数据库版本 1，以缓存 Promise 串行完成首次打开及建表。表约束类型和 `PARSED/FALLBACK_INVALID_RESPONSE` 状态，复合索引按事件时间、创建时间和 ID 倒序；无事件时间的卡片排在有事件时间卡片之后。初始化失败回滚，所有 ResultSet 均在 `finally` 关闭，生产适配器未新增 ArkTS 编译警告。
- 数据完整性与删除：事件时间保留原始带时区 ISO 文本，并转换为 epoch 毫秒用于跨时区正确排序；读取时重新校验枚举、精确实体结构、动作及事件时间/索引一致性，损坏或存储异常仅返回脱敏分类错误。更新只替换结构化理解和事件时间索引，不改变 ID、图片 URI 或创建时间；删除只删数据库行，不调用文件接口，因此不会删除相册源图、应用私有原图或缩略图。
- 链路接入：主页在理解成功或二次非法响应的安全 fallback 后立即创建卡片；认证、网络或服务错误没有理解结果，因此不落库。进入主页时读取已有卡片数量和最近保存 ID，临时验收区展示本地持久化摘要；落库失败不抹掉已经完成的 OCR、缩略图和结构化理解预览。正式时间轴、详情页和删卡 UI 仍留给 Task 2.4。
