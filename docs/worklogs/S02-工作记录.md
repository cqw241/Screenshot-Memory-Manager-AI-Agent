### Task 2.1 相册选图 + 端侧 OCR — [实现完成，真机验收待补]

- 实际做法（与计划的差异）：使用 `PhotoViewPicker` 单选图片；取消选择返回 `CANCELLED`。选中图片先通过只读文件描述符复制到 `filesDir/imports`，按识别到的 MIME 类型保留安全后缀，再从应用私有副本创建原始质量 `PixelMap` 并调用 Core Vision Kit `recognizeText()`。OCR 只统一换行并清理首尾空白；空结果或异常返回 `READY + RETRY_PENDING`。主页加入阶段性导入、处理中状态、私有副本信息和 OCR 预览，不含正式卡片。
- 发送确认：首次导入前按已验证配置的 `trustedOrigin` 说明 OCR 文本和缩略图的数据范围；确认记录存放于独立、非敏感 Preferences。服务源变化后必须重新确认。该步骤不读取或持有 API Key。
- 测试样例：在 `testdata/s02` 提交四张脚本确定性生成、无隐私的中文 PNG（讲座海报、取件码、DDL 通知、优惠券）及 `expected.json` 真值清单。
- 遇到的问题与解决：PhotoPicker 返回的媒体 URI 不能直接按普通路径复制；改为先 `fileIo.open()` 获取只读 fd，再由 fd 解码和复制。模拟器首次选择还会进入系统“仅可访问所选图片”确认页，这是系统隐私流程，不申请全相册读取权限。
- 自动验证：Hypium 36/36 通过；API 24 `assembleHap` 通过；`git diff --check` 通过。单元测试覆盖取消、成功、空 OCR、OCR 异常、文本规范化及资源释放。
- 模拟器验证：四张样例成功进入系统 PhotoPicker；讲座海报以 1080 × 1600、72158 bytes 完成应用私有复制。Core Vision 在当前 HarmonyOS Emulator 返回空文本，应用正确显示“待重试”且未崩溃。依既定口径，这不是 OCR 成功验收，四类 OCR 主要文字仍需 HarmonyOS NEXT 真机逐张复验。

### Task 2.2 LLM 理解客户端 — [实现完成，设备/真实模型验收待补]

- 实际做法（与计划的差异）：新增深理解模块，从 `filesDir/imports` 私有原图按比例生成 JPEG 85 缩略图并保存到独立的 `filesDir/thumbnails`；长边最大 1024 px，小图不放大。Base64 Data URL 只在缩略图生成器与模型请求内部传递，页面只接收缩略图 URI、宽高和编码字节数。主页现于单图导入和端侧 OCR 后继续发起理解，即使 OCR 为空也保留多模态图片理解，并在临时验收区展示类型、实体、动作与解析状态；正式卡片仍留待 Task 2.3–2.5。
- 请求与解析：调用必须经过 `ConfigRepository.withVerifiedConfig(...)`，并复用既有 URL 预检、可信源校验与 HTTP transport。Prompt 注入同一次捕获的设备本地时间、UTC 偏移和系统时区，要求保留 `timeText`、不得猜测缺失日期，并固定返回原始 JSON 对象。客户端精确校验对象字段、nullable string、类型/动作枚举及带时区且日历有效的 ISO-8601 `eventTime`；Markdown 围栏、语法错误、缺失/额外字段和错类型均为内容非法。首次非法使用同一 OCR、缩略图和时间上下文纠错一次，且不携带第一次原始回复；二次非法返回 `UNCATEGORIZED + 全 null 实体 + 空 actions + FALLBACK_INVALID_RESPONSE`。
- 安全与错误：每次请求对象的 Key 在 `finally` 清空，运行时 Key 再由 `withVerifiedConfig` 清空；Key、Base64、OCR 正文、完整请求体和模型原始回复不进入理解结果或日志。401/403、404/405、429、网络/超时、图片能力拒绝、其他 4xx/5xx 与外层响应不兼容分别返回脱敏分类错误，不生成未分类结果；未实现指数退避、后台重试或持久化队列。生产日志只记录缩略图宽高、编码字节数、请求次数、耗时与错误类别。
- 自动验证：Hypium 47/47 通过；覆盖横竖图缩放、小图不放大、缩略图请求、设备时间/时区注入、五种类型、严格非法矩阵、一次纠错、二次 fallback、错误分类和成功/异常/fallback 的 Key 清理。API 24 `assembleHap` 通过，`git diff --check` 通过；新增生产代码未引入编译警告，构建仍只有既有 HUKS、Preferences、OCR、PhotoPicker、私有原图关闭处理、`getContext` 弃用及未配置签名警告。
