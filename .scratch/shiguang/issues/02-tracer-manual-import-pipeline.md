# 02 — 核心曳光弹:手动导入截图 → OCR → LLM 结构化 → 时间轴出卡

**What to build:** 用户在 App 内从相册手动选择一张截图,系统完成:端侧 OCR(Core Vision Kit)提取文字 → OCR 文本 + 低分辨率缩略图按 vision 消息格式送云端多模态模型 → 返回固定 JSON schema 的结构化结果 `{类型, 关键实体(时间/地点/码号/金额), 可执行动作}` → 存入 RelationalStore → 时间轴页面出现结构化卡片。这是贯穿摄取/理解/记忆/呈现全层的第一条完整路径,后续所有票都在它之上生长。

**Blocked by:** 01 — 项目脚手架 + 首启配置向导

**Status:** ready-for-agent

- [ ] 手动导入一张讲座海报截图,时间轴出现类型正确、时间地点正确的卡片,附原图缩略图
- [ ] OCR 文本与缩略图同时送入模型(OCR 用于数字精度对照)
- [ ] LLM 返回非法 JSON 时自动重试一次,再失败落为"未分类截图"卡片,不崩溃
- [ ] 卡片数据持久化,重启 App 后时间轴仍在
- [ ] 4 类目标截图(海报/取件码/DDL 通知/优惠券)各导入一张,类型识别正确
